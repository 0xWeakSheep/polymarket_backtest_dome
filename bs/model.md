# 论文核心公式、理论价值计算与回测重点

本文档把重点放在这篇论文最关键的部分：理论价值如何定义、如何计算、哪些产品可以直接得到近似理论值、哪些必须数值求解，以及回测时应该验证哪些“理论值”。

---

## 1. 先说结论：这篇论文最核心的不是交易信号，而是理论价值框架

这篇论文真正想建立的是：

- 事件合约的基础理论价值定义
- 在 `log-odds` 状态变量下的一致定价内核
- 由这个内核推出的一整套衍生品理论价值计算方式

如果只抓一句话，全文最重要的意思是：

“事件合约的价格 `p_t` 必须是风险中性测度下的公平价值；为了保证这一点，作者不直接建模 `p_t`，而是建模 `x_t = ln( p_t / (1 - p_t) )`，并用鞅约束把 `x_t` 的漂移唯一确定下来，然后再对任意衍生品写出统一的定价方程。”

---

## 2. 论文里最核心的理论价值定义

### 2.1 基础事件合约的理论价值

基础事件合约在到期时只会支付：

- 事件发生：支付 `1`
- 事件不发生：支付 `0`

因此它在时刻 `t` 的理论价值定义为：

`p_t = E^Q[ 1_{事件发生} | F_t ]`

这条式子的地位最高。

它的含义是：
- `p_t` 既是市场价格
- 也是风险中性条件下的理论价值
- 同时还能被解释为风险中性概率

你可以把它理解成：

“这篇论文里所有更复杂的公式，最终都是为了让这个价格定义在动态模型下成立。”

### 2.2 为什么要引入 `x_t`

论文不直接建模 `p_t`，因为：

- `p_t` 只能在 `(0,1)` 内变化
- 靠近 `0` 和 `1` 时，直接建模容易数值不稳定

所以作者定义：

`x_t = ln( p_t / (1 - p_t) )`

反过来：

`p_t = 1 / (1 + e^(-x_t))`

记：

`S(x) = 1 / (1 + e^(-x))`

那么：

`p_t = S(x_t)`

所以论文真正做的是：
- 在 `x_t` 上建模
- 再映射回 `p_t`
- 最终让 `p_t` 仍然等于理论公平价值

---

## 3. 最核心的模型公式：公平价值约束如何锁定漂移

### 3.1 `x_t` 的动态

论文设：

`dx_t = μ(t,x_t) dt + σ_b(t,x_t) dW_t + ∫ z Ñ(dt,dz)`

这里：
- `μ(t,x_t)` 是漂移
- `σ_b(t,x_t)` 是信念波动率
- 跳跃项表示新闻或公告导致的突然重定价

### 3.2 为什么漂移不能随便设

因为基础价格必须满足：

`p_t = S(x_t)`

并且 `p_t` 在风险中性测度下必须保持鞅性质。

这就要求：

`E^Q[ p_u | F_t ] = p_t`

也就是说，价格作为理论公平价值，平均不能自己往上漂或往下漂。

### 3.3 漂移的约束公式

因此作者得到：

`0 = S'(x)·μ(t,x) + 1/2 · S''(x)·σ_b^2(t,x) + ∫[ S(x+z) - S(x) - S'(x)·χ(z) ] ν_t(dz)`

从而：

`μ(t,x) = - { 1/2 · S''(x)·σ_b^2(t,x) + ∫[ S(x+z) - S(x) - S'(x)·χ(z) ] ν_t(dz) } / S'(x)`

这条式子是全文最核心的理论创新。

它的真正价值是：

- 漂移不是自由参数
- 价格公平性先被定下来
- 模型能交易和能对冲的风险，剩下的是：
  - `σ_b(t,x)`：信念波动
  - `ν_t(dz)`：跳跃风险
  - 多事件相关性
  - 共跳结构

所以如果你问“论文最关键的公式是哪一个”，答案优先应该是这条漂移约束。

## 4. 一般理论价值公式：任意产品怎么定价

对任意终端支付 `g(x_T)`，产品理论价值 `V(t,x)` 满足：

`∂V/∂t + μ(t,x)·∂V/∂x + 1/2·σ_b^2(t,x)·∂^2V/∂x^2 + ∫[ V(t,x+z) - V(t,x) - (∂V/∂x)·χ(z) ] ν_t(dz) = 0`

终端条件：

`V(T,x) = g(x)`

这条 PIDE 是全文第二关键的理论价值公式。

它的含义是：

- 只要你给定到期支付结构 `g`
- 理论价值 `V(t,x)` 就应由这条方程决定

这相当于论文版本的“统一定价引擎”。

如果写成期望形式，也可以理解为：

`V(t,x) = E^Q[ g(X_T) | X_t = x ]`

在有路径依赖时，再扩展成对应的路径条件期望或边界问题。

## 5. 论文里几类产品的理论价值应该怎么理解

这一部分最关键，因为你后面不管做回测还是做图，本质都在验证这些理论值有没有解释力。

### 5.1 基础事件合约

理论价值：

`V_event(t) = p_t`

或者：

`V_event(t,x) = S(x)`

边界：
- `0 < p_t < 1`
- 实务上必须裁剪成 `p_t ∈ [ε, 1-ε]`
- 建议 `ε = 10^-5`

要验证的东西：
- 直接价格本身不是重点
- 重点是 `x_t` 的动态设定是否能让 `p_t = S(x_t)` 保持公平定价含义

### 5.2 信念方差互换

论文定义的核心对象之一，是 `x` 上的已实现二次变差：

`QV_x(t,T) = ∫ σ_b^2(u,x_u) du + Σ (Δx_u)^2`

它的理论公平执行价近似为：

`K_x-var(t,T) ≈ ∫ σ_b^2(u) du + ∫ λ(u) E[z^2(u)] du`

这条式子的意义非常大。

因为它说：

“方差产品的理论价值，本质上等于扩散方差贡献 + 跳跃二阶矩贡献。”

也就是说，理论价值不是拍脑袋给的，而是由模型里可估的两个风险源组成：
- 连续波动
- 跳跃风险

回测时最该验证的就是这条。

### 5.3 概率空间上的方差互换

若不是对 `x`，而是对 `p = S(x)` 定义方差，则短窗近似理论值为：

`K_p-var(t,t+Δ) ≈ [ p_t (1 - p_t) ]^2 · ∫ σ_b^2(u) du + ∫ [ S(x_t+z) - S(x_t) ]^2 ν_u(dz) du`

这里要特别注意：

- 接近 `p = 0` 或 `p = 1` 时，前面的 `[ p_t (1 - p_t) ]^2` 会变小
- 所以相同的 `x` 波动，对 `p` 的方差贡献会缩小

这正是论文里“为什么在 `x` 域建模更稳定”的一个理论原因。

### 5.4 相关性/协方差互换

两事件 `i, j` 的瞬时协方差近似为：

`Cov(dp_i, dp_j)_t ≈ S'_i S'_j σ_bi σ_bj ρ_ij(t) dt + ∫ Δp_i Δp_j ν_ij,t(dz_i,dz_j) dt`

它拆成两部分：

- 扩散相关贡献
- 共跳贡献

所以相关性类产品的理论价值，核心不是一个单独数字，而是：

“未来协方差或相关性执行价，应由平时联动和共同跳跃共同决定。”

回测时应重点验证：
- 扩散相关能解释多少
- 共跳能解释多少

### 5.5 Corridor Variance

这类产品只在指定概率区间内累计方差。

例如只在：

`p ∈ [a,b]`

时累计。

理论价值含义是：

“只给摇摆区间里的风险定价，不给边界附近几乎确定的区间定价。”

这是很有论文特色的一类产品，因为事件市场的风险确实主要集中在中间区域，而不是 `p` 已经接近 `0` 或 `1` 的地方。

理论计算上：
- 简单情况下可做冻结状态近似
- 一般情况下应通过带状态约束的 PIDE 计算

### 5.6 First-passage Note

这类产品不是看最终有没有发生，而是看：

“在 `T` 之前是否先触到某个概率阈值 `h`”

先映射成：

`x_h = ln( h / (1 - h) )`

理论价值本质上是：

“首达概率对应的风险中性期望值”

这里要注意：
- 有跳跃时，首达概率会明显改变
- 因为路径可以直接跳过阈值

所以这类产品是论文里体现“跳跃风险有定价意义”的重要例子。

## 6. 理论价值计算时必须明确的边界

这一部分必须单列，因为你后面不管是解释论文还是做图，边界不清楚就会把理论值算坏。

### 6.1 概率边界

由于：

`x = ln( p / (1 - p) )`

所以：
- `p = 0` 时发散
- `p = 1` 时发散

实务上必须使用：

`p ∈ [ε, 1 - ε]`

建议：
- 主方案：`ε = 10^-5`
- 噪音更大时：`ε = 10^-4`
- 不建议大于 `10^-3`

### 6.2 状态分区边界

为了理解理论价值在不同区域的含义，建议固定三段：

- 边界区：`p ∈ [ε, 0.1] ∪ [0.9, 1-ε]`
- 中间区：`p ∈ (0.1, 0.9)`
- 摇摆核心区：`p ∈ [0.25, 0.75]`

这些分区的理论意义分别是：
- 边界区：接近确定，`p` 空间对波动不敏感
- 中间区：价格最有信息量
- 摇摆核心区：风险最值得定价

### 6.3 时间边界

论文最强调的是短期限理论价值和短窗风险定价。

建议固定：
- 主窗口：`60 秒`
- 补充窗口：`300 秒`
- 扩展上限：`900 秒`

超过这个尺度后，很多短期跳跃和做市解释会被稀释。

### 6.4 跳跃边界

对跳跃风险理论值，至少要明确三种识别口径：

- 弱阈值：`|Δx| / σ̂_local >= 3`
- 标准阈值：`|Δx| / σ̂_local >= 4`
- 强阈值：`|Δx| / σ̂_local >= 5`

因为方差执行价里，跳跃二阶矩贡献对阈值设定很敏感。

### 6.5 相关性边界

对两事件的理论协方差价值，应按：

- 强负相关：`[-1, -0.5)`
- 弱负相关：`[-0.5, 0)`
- 弱正相关：`(0, 0.5]`
- 强正相关：`(0.5, 1]`

分别看结果。

否则你很难区分“扩散相关”和“共跳相关”的贡献。

## 7. 回测时真正要验证的，不是价格方向，而是这些理论价值

这一部分要改一下理解方式。

你不是主要去测：
- 做多赚不赚钱
- 做空赚不赚钱

你真正要测的是：

“论文给出的理论价值表达，能不能解释未来实现出来的风险和联动。”

### 7.1 第一优先级：验证方差执行价理论值

最该验证的是：

`K_x-var(t,T) ≈ 扩散贡献 + 跳跃贡献`

你应比较：
- 理论预测方差
- 未来真实实现方差

必须拿到：
- `MSE`
- `MAE`
- `QLIKE`
- `logMSE`（可选）

输出边界：
- 全样本
- 边界区
- 摇摆核心区
- 新闻窗口
- 非新闻窗口

### 7.2 第二优先级：验证跳跃价值贡献

要明确拿到：

`jump_share = 跳跃贡献 / 总理论方差`

并比较：
- 新闻窗口的 `jump_share`
- 非新闻窗口的 `jump_share`
- 边界区的 `jump_share`
- 摇摆核心区的 `jump_share`

这部分是在验证：

“论文把跳跃项单独写进理论价值里，到底是不是有实际解释力。”

### 7.3 第三优先级：验证协方差理论值

对事件对 `(i,j)`，验证：

`理论协方差 = 扩散相关贡献 + 共跳贡献`

至少输出：
- 扩散相关估计 `ρ_ij`
- 共跳频率
- `cojump_share = 共跳协方差贡献 / 总协方差`

### 7.4 第四优先级：验证边界区域下理论值是否仍稳定

必须看：
- 在 `p` 接近 `0` 或 `1` 时
- 用 `x` 域得到的理论值是否比 `p` 域更稳定

这是论文里非常核心的数值主张。

## 9. 最低可执行版本

如果你先只做一版最小可行分析，建议集中在下面三件事：

1. 基础理论价值框架  
   把 `p_t = S(x_t)`、漂移约束、PIDE 三件事讲清楚。

2. 方差执行价理论值  
   重点验证：
   `K_x-var ≈ 扩散贡献 + 跳跃贡献`

3. 协方差理论值  
   重点验证：
   `理论协方差 ≈ 扩散相关贡献 + 共跳贡献`

如果这三部分做扎实，这篇论文最核心的东西你就已经抓住了。

## 10. 你最终应该拿到的明确结论

做完后，至少要能回答下面这些问题：

1. 基础事件合约的理论价值定义是什么？
2. 为什么论文必须引入 `x = log-odds`，而不是直接在 `p` 上建模？
3. 为什么 `x_t` 的漂移不是自由参数，而是由公平定价条件唯一确定？
4. 方差产品的理论价值为什么等于扩散贡献加跳跃贡献？
5. 协方差产品的理论价值为什么等于扩散相关贡献加共跳贡献？
6. 为什么在边界区 `p ≈ 0` 或 `p ≈ 1` 时，直接在 `p` 空间做理论计算会更不稳定？
7. 论文提出的理论价值公式，是否真的能解释未来实现出来的方差、跳跃和联动？

如果这七个问题都能回答清楚，你对这篇论文的理解就已经抓到主干了。
